Role-Based Access Control (RBAC)

The tree_collaborators table had a role column (defaulting to editor) but no code enforced it — all collaborators had full write access regardless of role. This day's work introduced a complete RBAC system with granular ownership tracking.

Issue #83 → PR #121 — Enforce TreeCollaborator Role-Based Permissions Merged

  • Introduced two role levels: viewer (read-only) and editor (read/write)
  • Added roleFor(), canAccess(), and canEdit() methods to the FamilyTree model
  • 15 controllers modified — read endpoints use canAccess(), write endpoints use canEdit()
  • Relaxed BookExportController and BookSectionRevisionController from owner-only to canAccess() so collaborators can read
  • Relaxed AdvisorMessageController from owner-only to canEdit() so editors can send messages
  • Invitation system updated to accept optional role field (editor or viewer, defaults to editor)
  • New migration adds role column to invitations table
  • 25 new tests in CollaboratorRoleTest covering model methods, read/write enforcement, and invitation role propagation
  • All 176 existing tests pass with zero regressions

Issue #84 → PR #120 — Add Ownership Tracking & Scope Collaborator Writes Merged

  • Added nullable created_by UUID column to family_members, relationships, and life_events tables
  • Collaborators can only modify records they created; tree owners retain full write access
  • Auto-sets created_by on store operations; update/destroy check ownership via authorizeWrite()
  • Legacy records (created_by = null) protected from collaborator edits by design
  • Includes created_by in GDPR data export
  • 20 new tests in CollaboratorOwnershipTest covering create, edit, delete, owner override, and legacy record protection

Issue #85 → PR #119 — Hide Edit/Invite Forms from Collaborators Merged

  • Settings → Family tab previously showed edit/invite forms to all users, including collaborators who would get a 403 on submit
  • Collaborators now see a read-only view of tree details with a message: “You are a collaborator on this family. Only the owner can edit these details.”
  • Invite collaborator form completely hidden for non-owners
  • Uses current_user_role field from the tree list API response

  • Lazy-load tree-view: @xyflow/react (3.1 MB) + dagre (960 KB) loaded only when tree-view page is visited via next/dynamic with ssr: false

  • Removed cache: ‘no-store’ from client fetches (was preventing browser HTTP caching)
